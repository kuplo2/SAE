Présentation du groupe

Groupe 109 : Noémie JALIER, Léane CAILLEREZ, Daniel CHEN

Chaque mission de la SAE R1_04 possède différent objectif :

La première mission consiste à d'abord se familiariser avec les bases de données en analysant les données disponibles(sans requête SQL) dans le fichier excel de l'entreprise KDOU

La seconde mission nous amène à découvrir la base de donnée de celle-ci sous un fichier Access. C'est durant cette mission qu'il a fallu commencer à manipuler les requêtes SQL afin d'accéder aux données pertinents de l'entreprise.

La troisième mission comporte le même objectif que la seconde mais avec des requêtes plus complexes afin d'enrichir nos connaissances rudimentaire en SQL. Il a été notamment nécessaire d'employer des fonctions tel que GROUP BY, HAVING ou EXISTS afin de filtrer efficacement les données

Chaque requête ont été effectué et lancé directement avec l'IDE intégré d'Access qui ne possède aucune aide d'auto-complétion ni de détecteur d'erreur (comme à l'ancienne)

Mission 1

Le fichier mis a disposition lors de la mission 1 est un classeur excel contenant une feuille par table de la base de donnée de KDOU

Voici les différentes tables de cette base de donnée :

  • Fournisseur
  • Produit
  • Categorie
  • Clientele
  • Commande
  • DetailCommande

Pour résumer, lors de cette mission, on a vu que chaque table était conforme, c'est-à-dire qu'elles possédaient tous bien une clé primaire cependant on a pu observer diverses incohérences causé par l'absence de contraintes d'intégritées sur les tables.

Voici ci-dessous quelques exemples

Dans la table fournisseur, ces deux sociétés distincts possèdent le même N°Fournisseur(qui est une clé primaire), il y a une grave erreur

Dans la table Clientele, il y a deux clients qui possèdent le même numéro de téléphone, ce qui est absurde

Finalement, à travers l'analyse des données de KDOU sur le classeur excel, on peut dire avec certitude qu'ajouter des contraintes d'intégrités dans la base de donnée est nécessaire pour limiter les erreurs d'inattention. Ne pas utiliser excel comme outil de base de donnée en est une autre...
