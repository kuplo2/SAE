Mission 1

Le fichier mis a disposition lors de la mission 1 est un classeur excel contenant une feuille par table de la base de donnée de KDOU

Voici les différentes tables de cette base de donnée :

  • Fournisseur
  • Produit
  • Categorie
  • Clientele
  • Commande
  • DetailCommande

Pour résumer, lors de cette mission, on a veiller à vérifier que chaque table était conforme, c'est-à-dire qu'elles possédaient tous bien une clé primaire cependant on a pu observer diverses incohérences causé par l'absence de contraintes d'intégritées sur les tables.

Voici ci-dessous quelques exemples

Dans la table fournisseur, ces deux sociétés distincts possèdent le même N°Fournisseur(qui est une clé primaire), il y a une grave erreur

Dans la table Clientele, il y a deux clients qui possèdent le même numéro de téléphone (considéré comme une clé candidates), ce qui est absurde

Finalement, à travers l'analyse des données de KDOU sur le classeur excel, on peut dire avec certitude qu'ajouter des contraintes d'intégrités dans la base de donnée est nécessaire pour limiter les erreurs d'inattention. Ne pas utiliser excel comme outil de base de donnée en est une autre...
